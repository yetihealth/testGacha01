# 體態診斷卡 - 閃卡功能技術筆記

## 視覺效果總覽

### 1. 彩虹斜紋光澤(style-rainbow)
靠 `mix-blend-mode: color-dodge` 疊一層彩色斜紋,搭配 `background-position` 動畫讓光澤緩慢漂移。視覺最華麗,金屬感最強。

缺點:`mix-blend-mode` 在 html2canvas 截圖時相容性不穩定,顏色可能跟瀏覽器實際顯示不一致。如果最終要用 html2canvas 把卡片轉成圖片分享出去,這個風格有截圖跑掉的風險,正式上線前一定要實際測試一次。

### 2. 3D 視差傾斜光斑(style-tilt)
卡片本體用 `perspective` + `rotateX/rotateY` 做出立體傾斜,光斑用單純的 `radial-gradient` 疊圖,不靠混色模式。

優點:截圖相容性比彩虹斜紋穩定很多,因為沒有用到 `mix-blend-mode`。如果在意「截圖出去要好看」,這個是更保險的選擇。

**目前傾向採用這個風格作為正式版本。**

### 3. 開箱進場動畫(pulling-in)
每次抽卡時,卡片用 `scale + rotateZ` 做一個輕微的彈入效果,搭配一道掃光(`shine-sweep`)從左斜掃到右。這個是「抽卡瞬間」的核心反饋,比卡面本身閃不閃更重要,真正驅動使用者想一直抽的是這個瞬間的回饋感,不是卡面本身。

### 4. 稀有卡金色光暈脈衝(rare-glow)
抽到稀有卡時,卡片外圍會多一圈金色 `box-shadow` 脈衝,持續約 0.9 秒後消退,用來強化「這次不一樣」的瞬間感。

### 5. 粒子噴發(spark)
抽到稀有卡時,額外從卡片中心噴出約 14 個小圓點向四周飛散並淡出,純 CSS keyframe 動畫,沒有用任何外部函式庫。

## 互動輸入方式

### 滑鼠移動(桌機)
監聽 `mousemove`,算出滑鼠在卡片內的相對位置(百分比),拿這個位置去驅動光斑座標跟傾斜角度。

### 觸控拖曳(手機)
監聽 `touchmove`,抓 `touches[0]` 的座標,套用跟滑鼠完全相同的 `applyPointer()` 函式。手機跟桌機共用同一套運算邏輯,行為一致。

**目前決定:正式版本只做這個,不做陀螺儀。**

### 陀螺儀傾斜()
監聽 `deviceorientation`,用手機物理傾斜角度(`beta`/`gamma`)驅動卡片旋轉,不用碰卡片本身就會反光,手感最接近真實閃卡。

技術限制:
- iOS 13+ 要求必須由使用者主動點擊按鈕觸發 `DeviceOrientationEvent.requestPermission()` 才能讀取角度資料,這是系統隱私規範,無法繞過。
- 只能在 HTTPS 環境下生效,`file://` 本機測試會被部分瀏覽器擋掉。
- 用的是「相對角度」校正,以第一次讀到的角度當基準點再算偏移量,這樣不管使用者拿手機的姿勢是什麼,傾斜手感都一致。

這個版本的程式碼留存在 demo 檔案裡,如果之後想重新評估,可以直接拿來用,不用重新寫。

## 稀有度機制

**設計決策:稀有度是完全獨立的隨機判定,跟診斷出來的代償類型內容無關。**

```js
const isRare = Math.random() < RARE_CHANCE;
```

抽到哪種代償類型,跟抽不抽到稀有卡,是兩條互不影響的邏輯。`RARE_CHANCE` 在 demo 裡為了方便測試調到 0.3,正式上線前要調回 0.01~0.03(1%~3%)左右,數值愈低,抽到的瞬間愈有稀有感。

## Token 成本說明

閃卡的所有視覺效果(漸層、動畫、傾斜、粒子)都是純前端 CSS/JS,在使用者瀏覽器裡運算跟渲染,**不會呼叫任何 AI API,跑幾百次都是零 token 成本**。

唯一會耗 token 的,是診斷卡裡「用 Claude API 生成個人化文字」那一段,跟卡面的視覺特效是兩件完全分開的事,不會互相影響預算。

## html2canvas 截圖注意事項

正式串接進你的 Canvas/html2canvas 渲染流程之前,務必實際截一張稀有卡測試:

1. 如果用彩虹斜紋風格,先確認 `mix-blend-mode` 截圖出來的顏色跟瀏覽器顯示一致。如果跑掉,改用 3D 視差光斑風格(不靠混色模式),或把截圖前那一刻的疊圖換成單純半透明漸層。
2. 動畫效果在截圖瞬間會被定格,這是合理的,分享出去的圖會是當下那個光澤角度,不會動。
3. 3D 視差的 `transform`(`rotateX`/`rotateY`)html2canvas 支援度比混色模式好,風險較低。
